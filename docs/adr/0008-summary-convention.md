# 0008: 先頭コメントによるセル要約の規約

- Status: Accepted
- Date: 2026-07-02

## Context

`list_cells` の一覧から notebook 全体の構造を素早く把握したい。従来は先頭1行を truncate した
`source_preview` だけで、コードセルの意図が読み取りにくかった。人間・AI 双方にとって
「各セルが何をするか」を軽量に一覧化したい。

## Decision

**セル先頭の連続する `#` コメント行**をそのセルの要約とみなす規約を導入する。

- code: 先頭から連続する `#` 行を要約に採用。最初の非コメント行（空行含む）で打ち切り。
  コメントが無ければ先頭非空行の1行プレビューにフォールバック。
- markdown/raw: 先頭の非空行を採用。
- cap: 最大 3 行 / 各行 100 字。
- `list_cells` は `source_preview` を廃し `summary`（複数行可）を返す。
- insert/edit の tool description に「コードセルは先頭に `#` 要約を付ける」と明記し、
  クライアント（AI エージェント）がこの規約に沿ってメタデータを書くよう誘導する。

## Consequences

- 一覧1回で目次相当が得られ、構造把握が速い。
- ツールが規約を教えるため、運用が自然に回る（別途ドキュメントの徹底が不要）。
- 破壊的変更: `list_cells` の `source_preview` → `summary`（pre-1.0 のため許容）。
- 制約: 要約はあくまで先頭コメント依存。付いていないセルは1行プレビューに留まる。

## Alternatives considered

- **専用 `outline` ツールを新設**: ツールが増え ADR-0001 の薄さに反する。list_cells 拡張に統一。
- **セル metadata に要約フィールドを持たせる**: notebook を汚し、他ツールと非互換。ソース内コメントを採用。
